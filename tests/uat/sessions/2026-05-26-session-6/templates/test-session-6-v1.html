Pre-session setup (do once before starting):
1. Deploy the orchestrator image at HEAD of uat/session-6 to the lancache host (or run locally if you prefer).
2. Confirm ORCH_TOKEN, ORCH_DATABASE_PATH, and ORCH_STEAM_SESSION_DIR are set per your deployment.
3. Start the service: uvicorn orchestrator.api.main:app --host 127.0.0.1 --port 8765.
4. Wait for api.boot.complete in the logs before running any scenario.
Token shorthand: all curl examples assume $TOK holds your bearer (export TOK="$ORCH_TOKEN").

Feature 10: BL10 — Steam authentication substrate

Endpoints under test:
POST /api/v1/platforms/steam/auth — initial auth, returns 200 (no 2FA) or 202 + challenge_id
POST /api/v1/platforms/steam/auth/{challenge_id} — 2FA completion
GET /api/v1/platforms/steam/auth/status — current session state
GET /api/v1/platforms — read-only platforms row including auth_status
SEV-2 fix under validation: F-UAT6-2 (worker reads ORCH_STEAM_SESSION_DIR env var instead of hardcoded path).

Feature 11: BL11 — Steam library sync

Endpoints under test:
POST /api/v1/platforms/steam/library/sync — manual trigger, returns 202 + job_id
GET /api/v1/jobs?kind=library_sync — see queued/running/completed sync jobs
GET /api/v1/games?platform=steam — verify games table populated
SEV-2 fixes under validation:
  F-UAT6-1 (large-library response no longer hits asyncio's 64 KiB readline cap)
  F-UAT6-3 (session-expiry mid-sync flips platforms.auth_status='expired').
